### [SH-4002] Especificación: Shell Orchestration & GHCR Publishing (Zero-Build)

#### 1. Visión General (Abstract)
En paridad con el Kernel [ANK-4001], el despliegue del "Thin Client" (Aegis-Shell) debe transicionar a `Pull-on-Deploy` para entornos Edge. La imagen de Docker (FastAPI + React/Vite) será pre-compilada y subida a GitHub Container Registry.

#### 2. Pipeline de Publicación (`.github/workflows/docker-publish.yml`)
- **Triggers:** Push a `main` y tags de versión.
- **Autenticación (GHCR):** Uso de `GITHUB_TOKEN` (`packages: write`).
- **Construcción (Build & Push):** 
  - Extracción de tags con `docker/metadata-action`.
  - Construcción del `Dockerfile` (BFF y UI).
  - Empuje a `ghcr.io/<owner>/aegis-shell:latest`.

#### 3. Política Zero-Panic SRE
- El nombre de la imagen debe ser forzado a minúsculas para cumplir con el estándar de Docker/GHCR.
- Uso de caché de capas (`type=gha`) para optimizar tiempos de compilación de `npm install` y `pip install`.
