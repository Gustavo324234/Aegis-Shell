###[SH-120] Especificación: Frontend Audio Capture (Siren Web API)

#### 1. Visión General (Abstract)
El ecosistema Aegis ya posee un canal bidireccional estable desde el BFF hacia el Kernel para el Protocolo Siren. Este ticket aborda la capa de presentación: capturar la voz del usuario, pre-procesarla y enviarla al BFF minimizando la latencia.
Para proteger los recursos de CPU del Kernel, el frontend debe enviar **Raw PCM a 16kHz, 16-bit, Mono**. 

#### 2. Arquitectura de Audio (Zero-Overhead Processing)
- **AudioContext:** Se inicializará un contexto de audio forzando `sampleRate: 16000`.
- **Procesamiento:** Utilizaremos un `ScriptProcessorNode` o `AudioWorkletNode` para capturar buffers de audio crudos en tiempo real (ej. chunks de 250ms o 512/1024 frames).
- **Conversión:** Los Float32Arrays nativos de la Web Audio API se convertirán a Int16Array (PCM 16-bit) en el hilo del navegador (o en un Web Worker/Worklet para no bloquear el renderizado de React).

#### 3. Gestión de Estado (Zustand)
- Se añadirá a `useAegisStore.ts` un nuevo estado: `isRecording: boolean`.
- Acciones: `startSirenStream()` y `stopSirenStream()`.
- **WebSocket Dedicado:** El streaming de voz no debe multiplexarse sobre el WS de chat existente para evitar bloqueos por grandes payloads de texto. Se abrirá una conexión limpia a `ws://localhost:8000/ws/siren/{tenant_id}?session_key=...`.

#### 4. Interfaz de Usuario (UI)
- En `ChatTerminal.tsx`, añadir un botón de Micrófono (`Mic` icon de Lucide) junto al botón de enviar.
- El botón tendrá un diseño "Hold to Talk" (Push-To-Talk) o "Toggle". Cuando `isRecording` sea `true`, el Orbe central o el botón deben mostrar feedback visual (animación de pulso) usando Framer Motion.

#### 5. Criterios de Aceptación (DoD)
1. **Audio Pipeline:** Captura de micrófono con downsampling a 16kHz, Mono, Int16 PCM.
2. **WebSocket Lifecycle:** El WS a `/ws/siren/` se abre al iniciar la grabación y se cierra limpiamente al detenerla. Se manejan errores de conexión (`onError`, `onClose`) actualizando la UI.
3. **Resiliencia de Hardware:** Si el usuario deniega permisos de micrófono, capturar la excepción de `getUserMedia` y mostrar un toast/mensaje de error elegante sin crashear la aplicación React.
