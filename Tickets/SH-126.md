###[SH-126] Especificación: The Engine Setup Wizard (Hardware Aware)

#### 1. Visión General (Abstract)
Tras crear su usuario Master en la pantalla "Zero-Trust", el usuario debe ser recibido por un Asistente Visual (Wizard) que detecte su hardware y le recomiende el motor cognitivo adecuado, permitiendo ingresar la API Key cómodamente desde el navegador.

#### 2. BFF Passthrough (`main.py`)
- Crear el endpoint HTTP POST `/api/engine/configure` que reciba el JSON de configuración de la UI y llame al nuevo RPC `ConfigureEngine` del Kernel.

#### 3. Detección de Hardware & UI (React)
- Crear `EngineSetupWizard.tsx`. Esta pantalla intercepta al usuario antes de dejarlo entrar al Chat si no hay un motor configurado.
- **Hardware Awareness:** Llama a `GetSystemStatus`. Si `vram_total_mb` es menor a 8000 MB (8GB), la UI muestra una alerta SRE: *"Hardware Limitado Detectado. Recomendamos usar el Motor Cloud (API) para evitar degradación de rendimiento."*
- **Selector:** Dos tarjetas inmersivas (Glassmorphism):
  1. `[Local Inference]`: (Desactivado o advertencia si hay poca VRAM).
  2. `[Cloud Inference]`: Pide URL de la API, Modelo y API Key (Oculta con asteriscos).
- Al enviar, hace POST al BFF y avanza al chat.

#### 4. Criterios de Aceptación (DoD)
1. Endpoint BFF implementado y enlazado al gRPC de ANK.
2. `EngineSetupWizard` creado con renderizado condicional basado en la memoria VRAM real del sistema.
3. El estado de configuración fluye correctamente hasta la base de datos cifrada de Ring 0.
