### [SH-124] Especificación: SRE Bootstrap UX Redesign

#### 1. Visión General (Abstract)
La pantalla actual de inicialización (`AdminSetupScreen.tsx`) causa fricción cognitiva. Los usuarios confunden la fase de "Creación del Master Admin" con un "Login", esperando credenciales por defecto. Siguiendo las directrices Zero-Trust, Aegis no posee credenciales por defecto. Se rediseñará la UX para dejar explícitamente claro que el sistema está "virgen" y que el usuario está forjando la primera llave de acceso.

#### 2. Cambios de Interfaz (Copywriting & Layout)
- **Título de Pantalla:** Cambiar de "AEGIS BOOTSTRAP" a "INICIALIZACIÓN ZERO-TRUST".
- **Banner Informativo:** Añadir una caja de texto destacada (estilo alerta SRE) que diga: *"Sistema limpio detectado. Por directiva de seguridad militar, Aegis OS no posee contraseñas por defecto. Forja tu credencial de Master Admin ahora."*
- **Labels de Formularios:** Cambiar los textos estáticos para denotar acción de creación:
  - "Master Admin ID" -> "Crear ID de Administrador (ej. root)"
  - "Master Passphrase" -> "Nueva Contraseña Maestra"
  - "Confirm Passphrase" -> "Repetir Contraseña"
- **Botón de Acción:** Cambiar de "Initialize Identity" a "Forjar Llaves y Bloquear Sistema".

#### 3. Autocompletado Asistido
- Para agilizar el proceso sin romper la seguridad, el campo de ID de Administrador puede venir pre-rellenado con la palabra `root`, permitiendo al usuario solo enfocarse en escribir una contraseña segura dos veces.

#### 4. Criterios de Aceptación (DoD)
1. Modificar `ui/src/components/AdminSetupScreen.tsx`.
2. Incluir el banner de explicación Zero-Trust.
3. Modificar todos los labels y placeholders para que transmitan "Creación" y no "Autenticación".
4. Verificación visual para asegurar que mantiene la estética Cyber-Minimalist.
