### [SH-125] Especificación: Seamless File Injection & Workspace Sync

#### 1. Visión General (Abstract)
La UX de Aegis Shell debe ser "Zero-Code" para el usuario final. En lugar de mapear volúmenes de Docker manualmente, la Shell permitirá la inyección dinámica de archivos al `Workspace` del Kernel mediante la interfaz gráfica (Drag & Drop y Botón de Adjunto), respaldado por un endpoint de ingesta en el BFF.

#### 2. Endpoint de Ingesta (BFF - FastAPI)
- Crear un nuevo endpoint HTTP POST `/api/workspace/upload` en `bff/main.py`.
- **Seguridad Citadel:** El endpoint DEBE requerir `tenant_id` y `session_key`. Validará las credenciales contra el Kernel mediante un chequeo rápido (`get_system_status` o similar).
- **Escritura Segura:** FastAPI guardará el archivo recibido (usando `UploadFile`) directamente en el directorio montado del tenant: `../users/{tenant_id}/workspace/{filename}`.

#### 3. Interfaz de Usuario (React)
- **Drag & Drop:** Implementar una zona de "Drop" global o en el área de input del `ChatTerminal.tsx`.
- **Botón de Adjunto:** Un icono de "Clip" (`Paperclip` de Lucide) al lado del botón de micrófono. Al hacer clic, abre el selector de archivos del sistema.
- **Feedback Visual:** Al soltar o seleccionar un archivo, mostrar un indicador de carga en el input. Una vez que el BFF confirme la recepción, insertar un mensaje del sistema en el chat: *"Archivo [nombre] inyectado en el Workspace"*, para que el usuario sepa que ya puede pedirle cosas a la IA sobre él.

#### 4. Criterios de Aceptación (DoD)
1. Endpoint `/api/workspace/upload` implementado con validación Citadel.
2. Botón de carga en la UI funcionando.
3. El archivo subido desde el navegador aparece físicamente en la carpeta `/app/users/{tenant_id}/workspace` del contenedor Docker.
4. El proceso es asíncrono y no congela la UI ni interrumpe el WebSocket de voz/chat.
